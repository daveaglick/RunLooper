bool RunLooper.RunLoop.Yield ( bool  block)

Allows the RunLoop to continue processing events. This can only be called from the main thread (calling from another thread will always return false). This method should be used if the caller is on the main thread but needs to wait for another thread. Without calling this while waiting, the main thread will block for an extended period and events will pile up.

Parameters:
blockIndicates if the calling thread should block until an event is available to process.
Returns:
True if an event was processed (will always return true if block is true unless the RunLoop is disposed).

Allows the RunLoop to continue processing events until a specified AutoResetEvent is signaled or until a specified TimeSpan is reached (whichever comes first). This method should be used if the caller is on the main thread but needs to wait for another thread. Returns:
true if the AutoResetEvent was signaled.
